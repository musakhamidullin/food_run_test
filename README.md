# Тестовое задание: Flutter-разработчик — FoodRun

**Компания:** FoodRun — сеть ресторанов быстрого питания (iOS + Android)
**Проект:** мобильное приложение клиента, production, ~180 Dart-файлов
**Стек:** Flutter 3.22 · Dart 3.4 · GetX 4.6 · `http` · `hive_ce` · `shared_preferences` · `web_socket_channel` · Firebase FCM · Sentry

---

## Контекст

Перед вами исходный код мобильного приложения сети ресторанов.
Проект работает в production. Приложением пользуется 1000 человек ежедневно.
Вам предстоит работать с этими исходниками.

Ваша задача: разобраться, предложить конкретный план своего онбординга и дальнейших улучшений.

В этом репозитории лежат реальные файлы проекта (частично изменённые). Читайте не только
`lib/` — смотрите весь репозиторий.

Архитектурный обзор:

```
lib/
├── main.dart                        — точка входа, инициализация приложения
├── firebase_options.dart            — конфигурация Firebase (FlutterFire CLI)
├── network/
│   ├── http_client.dart             — HTTP-клиент (GET, POST, PATCH, DELETE)
│   ├── apis.dart                    — все API-методы приложения
│   ├── ws_client.dart               — WebSocket-клиент
│   └── api_exception.dart           — типизированные исключения
├── controllers/
│   ├── auth_controller.dart         — авторизация по SMS
│   ├── menu_controller.dart         — меню, категории, продукты
│   └── order_controller.dart        — корзина, скидки, бонусы, оформление заказа
├── services/
│   ├── order_service.dart           — бизнес-логика корзины
│   └── order_status_mapper.dart     — маппинг статусов заказа для UI
├── models/                          — модели данных
├── components/                      — переиспользуемые виджеты
│   ├── rating_tag_button.dart
│   ├── rating_tags_var1.dart
│   ├── rating_tags_var2.dart
│   └── rating_tags_var3.dart
└── screens/
```

## Задания

### 1. Аудит репозитория

Изучите **весь репозиторий**. Составьте структурированный список найденных проблем.

---

### 2. Рефакторинг `HttpClient`

Перепишите `lib/network/http_client.dart`, устранив все найденные проблемы.

---

### 3. Архитектурный разбор `OrderController`

`order_controller.dart` — это реальный файл из production-проекта.

1. Какие проблемы вы в нём видите? Перечислите их и объясните, в чём именно проблема и насколько она серьёзная.
2. Предложите конкретный план рефакторинга: на какие классы вы бы разделили этот контроллер? Что в каждый входит?
   Нарисуйте схему зависимостей (текстом или ASCII).
3. В `menu_controller.dart` есть баг, который проявляется при нестабильном соединении.
   Найдите и опишите его: как воспроизвести, что увидит пользователь.

Запишите ответ в виде скринкаста.
